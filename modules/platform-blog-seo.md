## Nhiệm vụ

Viết bài blog/article tối ưu SEO, rank Google, vừa hay cho người đọc vừa tốt cho bot.

## Blog SEO Rules 2025

### Format
```
- Độ dài: 1000-3000 từ (sweet spot: 1500-2200 cho VN)
- Heading: H1 (1 duy nhất) → H2 (3-7) → H3 (tùy)
- Paragraph: 2-4 câu/đoạn (scannable)
- Visual: 1 hình/bảng mỗi 300-500 từ
- Internal links: 3-5 links tới bài liên quan
- External links: 2-3 links tới nguồn uy tín
```

### SEO On-Page Checklist

```
☐ Target keyword trong Title (H1) — gần đầu càng tốt
☐ Target keyword trong first 100 words
☐ Target keyword trong 1-2 H2
☐ Meta title: ≤60 chars, có keyword + benefit
☐ Meta description: ≤155 chars, có keyword + CTA
☐ URL slug: ngắn, có keyword, no accent (SEO tiếng Việt)
☐ Image alt text: mô tả + keyword tự nhiên
☐ Related keywords / LSI: rải tự nhiên trong body
☐ Table of Contents: cho bài >1500 từ
```

### Featured Snippet Optimization

```
Dạng 1: Paragraph snippet (trả lời trực tiếp trong 40-60 từ)
Dạng 2: List snippet (ordered/unordered list, 5-8 items)
Dạng 3: Table snippet (comparison table, data table)

→ Đặt câu trả lời NGAY SAU H2/H3 heading dạng câu hỏi
→ Format rõ ràng: bold, list, table
```

### Content Types & Keyword Intent

| Search Intent | Content Type | Ví dụ keyword |
|--------------|-------------|---------------|
| Informational | How-to, Guide | "cách viết CV", "hướng dẫn SEO" |
| Navigational | Brand page | "Shopee login", "Grab đăng ký" |
| Commercial | Comparison, Review | "so sánh iPhone 16 vs 15", "review máy lọc" |
| Transactional | Landing page | "mua laptop giá rẻ", "đăng ký khóa học" |

### Internal Linking Strategy

```
1. Pillar-Cluster: Bài pillar dài link → bài cluster chi tiết
2. Contextual: Link từ trong câu, anchor text tự nhiên
3. Tránh: "Click vào đây" → Dùng: "đọc thêm về [keyword]"
4. Số lượng: 3-5 internal links / 1500 từ
5. Ưu tiên: Link bài có traffic cao + bài mới cần boost
```

## Anti-AI Rules cho Blog SEO

```
1. KHÔNG: Keyword stuffing — tự nhiên hóa keyword placement
2. PHẢI: Intro có personality — không generic overview
3. PHẢI: Ví dụ thực tế VN (tên tool, brand, case study VN)
4. Tránh: "Conclusion" / "In conclusion" — viết kết tự nhiên
5. FAQ: Trả lời như TƯ VẤN, không như từ điển
6. Xen kẽ: Opinion cá nhân + data → không 100% factual dry
```
