## Nguyên Tắc Thiết Kế Slide

### Rule 1-6-6 (Chuẩn vàng)

```
- 1 ý chính / slide
- ≤6 bullets / slide
- ≤6 từ / bullet (tiếng Việt: ≤10 từ)
- Nếu vượt → TÁCH slide
```

### Visual Hierarchy (Thứ tự mắt đọc)

```
MẮT ĐỌC THEO:
┌─────────────────────────┐
│ TITLE (lớn, bold)   ←── │ Đọc đầu tiên
│                         │
│ ● Key point 1       ←── │ Đọc thứ 2-4 (scan)
│ ● Key point 2           │
│ ● Key point 3           │
│                         │
│ [Visual element]    ←── │ Hỗ trợ, không thay thế text
│                         │
│ Source: [nhỏ, góc dưới] │ Đọc cuối (nếu cần)
└─────────────────────────┘
```

### Contrast & Readability

```
Font size tối thiểu:
- Title: ≥28pt
- Body: ≥20pt
- Notes/source: ≥14pt

Contrast:
- Dark text trên light background (formal, daytime)
- Light text trên dark background (dramatic, evening)
- KHÔNG: văn bản trên hình nền busy
- PHẢI: text-shadow hoặc semi-transparent overlay nếu text trên ảnh

Typeface:
- Sans-serif cho body (Roboto, Open Sans, Montserrat)
- Max 2 fonts / deck
- Bold cho emphasis, KHÔNG underline
```

## Narrative Arc Patterns — Chi Tiết

### Arc 1: Problem → Solution (Business)

```
ACT 1 — SETUP (20% slides)
│ Hook: Số liệu shock / tình huống thật
│ Problem: Đào sâu pain point, ai bị ảnh hưởng?
│ Stakes: Nếu không giải quyết → hậu quả gì?

ACT 2 — CONFRONTATION (50% slides)
│ Analysis: Root cause — tại sao vấn đề tồn tại?
│ Options: Các cách tiếp cận (loại bỏ yếu)
│ Solution: Framework/giải pháp đề xuất
│ How it works: Step-by-step
│ Evidence: Case study, data, demo

ACT 3 — RESOLUTION (30% slides)
│ Results: Before/After, ROI
│ Next steps: Action plan cụ thể
│ CTA: Yêu cầu hành động rõ ràng
│ Q&A
```

### Arc 2: Minto Pyramid (McKinsey-style, Executive)

```
Slide 1: ANSWER FIRST — kết luận/recommendation ngay
Slide 2-3: KEY ARGUMENTS — 3 lý do chính (MECE)
Slide 4-9: SUPPORTING DATA — evidence cho từng argument
Slide 10: SO WHAT — implications, action items

Rule: Executive KHÔNG có thời gian nghe build-up
→ Answer first, justify after
```

### Arc 3: Teaching Arc (Giảng dạy)

```
PRIOR KNOWLEDGE: Nhắc lại bài trước (1-2 slides)
HOOK: Câu hỏi/demo/video gây tò mò (1 slide)
OBJECTIVE: Hôm nay sẽ học gì (1 slide)
CONTENT: Lý thuyết + ví dụ xen kẽ (5-10 slides)
PRACTICE: Bài tập/activity (2-3 slides)
SUMMARY: Key takeaways (1 slide)
PREVIEW: Bài sau sẽ học gì (1 slide)
```

### Arc 4: TED-style (Inspiration)

```
STORY OPENING: Kể 1 câu chuyện cá nhân (2 slides)
INSIGHT: Bài học rút ra — "big idea" (1 slide)
EVIDENCE: 3 examples/studies hỗ trợ (4-6 slides)
PERSONAL: Quay về story, deepening (1-2 slides)
CALL TO ACTION: Khán giả nên làm gì? (1 slide)

Rule: Emotion > Logic cho TED
→ Story chiếm ≥40% thời lượng
```

## Chart Selection Guide

### Chọn Biểu Đồ Phù Hợp

| Muốn show | Dùng chart | Ví dụ |
|-----------|-----------|-------|
| **So sánh** | Bar chart (ngang/dọc) | Revenue theo quý, so sánh đối thủ |
| **Xu hướng** | Line chart | Tăng trưởng 5 năm, traffic trends |
| **Tỷ lệ/Phần** | Pie / Donut chart | Market share, budget allocation |
| **Phân bố** | Scatter plot | Correlation giá-chất lượng |
| **Quy trình** | Flow chart | Pipeline, workflow, decision tree |
| **Timeline** | Timeline / Gantt | Lịch sử, project plan |
| **So sánh 2 option** | Table / Side-by-side | Feature comparison |
| **1 số lớn** | Big Number slide | "73%" full slide |
| **Quan hệ** | Venn diagram / Matrix | Overlap, positioning |

### Chart Rules

```
1. MỖI chart = 1 insight rõ ràng — không nhồi data
2. Title chart = KEY INSIGHT (không mô tả data)
   ❌ "Revenue 2020-2025"
   ✅ "Revenue tăng gấp 3 sau chiến dịch Q3/2023"
3. Highlight: Tô đậm / khác màu phần muốn nhấn
4. Source: Luôn ghi nguồn data
5. Label: Đủ label để hiểu mà KHÔNG cần xem legend
```

## Speaker Notes Best Practices

### Speaker Notes ≠ Đọc Slide

```
SLIDE nói GÌ:     "Revenue tăng 40% YoY"
SPEAKER NOTE nói:  "Để tôi kể chuyện đằng sau con số 40% này.
                    Đầu Q3 chúng tôi suýt cancel cả chiến dịch vì budget bị cắt.
                    Nhưng team marketing quyết định pivot sang social-first..."

→ Slide = WHAT
→ Speaker = WHY / HOW / STORY
```

### Template Speaker Notes

```
1. TRANSITION: "Từ [slide trước], chúng ta chuyển sang..."
2. CONTEXT: "Tại sao điều này quan trọng: ..."
3. STORY/EXAMPLE: "Để tôi chia sẻ 1 case thú vị..."
4. DATA STORY: "Con số này cho thấy..."
5. PAUSE CUE: "[Để khán giả phản ứng 3 giây]"
6. INTERACTION: "Giơ tay nếu bạn đã gặp tình huống này"
7. NEXT: "Và điều đó dẫn chúng ta tới..."
```

## Vietnamese Presentation Culture

### Văn Hóa Thuyết Trình VN

```
1. HIERARCHY: Kính trọng người nghe senior — mở đầu chào, cảm ơn đúng cách
2. SAVING FACE: Criticism luôn constructive, không attack trực tiếp
3. GROUP HARMONY: Nhấn mạnh team effort, collective benefit
4. DATA TRUST: Người VN tin data hơn opinion — luôn có số liệu
5. STORY POWER: Storytelling cá nhân rất hiệu quả ở VN
6. Q&A: Hay có câu hỏi "sếp hỏi" — chuẩn bị kỹ slides backup
7. TIME: Thường chạy overtime — plan 10% buffer
```

### Do & Don't

```
DO:
✅ Chào hỏi đúng cách (kính gửi Quý vị...)
✅ Case study VN, data VN
✅ Humor nhẹ, phù hợp (self-deprecating OK)
✅ Kết thúc đúng giờ hoặc sớm hơn
✅ Handout / takeaway cho người nghe

DON'T:
❌ Đọc slide word-by-word
❌ Quá nhiều text, font nhỏ
❌ Slide tiếng Anh 100% cho audience VN
❌ Humor nhạy cảm (chính trị, tôn giáo, vùng miền)
❌ Nói "Slide tiếp theo" — dùng transition tự nhiên
```

## Slide Type Library

### Essential Slides

```
1. TITLE SLIDE: Tên + Subtitle + Tên người trình bày + Ngày
2. AGENDA: Table of Contents, set expectations
3. SECTION DIVIDER: Full-screen, 1 phrase, pause moment
4. BIG NUMBER: 1 stat lớn + 1 câu giải thích
5. QUOTE: Trích dẫn expert + source
6. COMPARISON: 2 columns side-by-side
7. TIMELINE: Chronological progression
8. TEAM SLIDE: People + roles + photos
9. THANK YOU + Q&A: Contact info, CTA
10. BACKUP SLIDES: Data deep-dive cho những câu hỏi khó
```
